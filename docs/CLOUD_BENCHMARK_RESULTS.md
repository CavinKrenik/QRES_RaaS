## 2. System Performance: Swarm vs. Federated Learning

Comparing **QRES (UDP/Gossip)** against **Federated Averaging (TCP/Centralized)** on a simulated LoRaWAN link (56 kbps, 15% Packet Loss).

### The "Bandwidth Gap"

The core innovation is transmitting "Evolved Genes" (Bytecode) instead of "Gradient Vectors" (Floats).

| Metric | FedAvg (Baseline) | QRES (Swarm) | Improvement |
| :--- | :--- | :--- | :--- |
| **Payload Size** | 2.4 MB (Model) | 1.6 KB (Gene) | **1500x Smaller** |
| **Daily Bandwidth** | 480 MB | **8 KB** | **Orders of Magnitude** |
| **Convergence** | 14.5 Hours | **1.2 Hours** | **12x Faster Wall-Clock** |

### Resilience to Network Failure

At 15% packet loss, TCP-based Federated Learning suffers from Head-of-Line blocking.

| Packet Loss | FedAvg Success | QRES Success | Notes |
| :--- | :--- | :--- | :--- |
| 0% (ideal) | 100% | 100% | Both succeed when pipes are clean |
| 5% (Wi-Fi) | 85% | 100% | TCP HoL blocking penalizes FedAvg |
| 15% (mesh) | 12% (timeouts) | 98% | UDP gene gossip shrugs off drops |

### Conclusion

QRES trades local compute time (21ms encoding vs 2.9ms) for network survival. In edge environments where bandwidth is the scarce resource, this trade-off is the difference between a working system and a timeout.
